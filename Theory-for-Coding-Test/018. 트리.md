## 1. 트리(Tree)

- 노드들이 계층적으로 연결된 비선형 자료구조
- 하나의 루트 노드에서 시작하여 여러 하위 노드로 분기되는 구조를 가짐
- 대표적인 예 : 컴퓨터의 디렉토리 구조

---

## 트리의 기본 용어

- 노드(Node): 트리의 기본 단위로, 데이터와 하위 노드에 대한 포인터를 포함
- 간선(Edge): 노드 간의 연결을 나타내는 선
- 루트 노드(Root Node): 부모가 없는 최상위 노드입
- 부모 노드(Parent Node): 자식 노드를 가진 노드
- 자식 노드(Child Node): 부모 노드의 하위 노드
- 형제 노드(Sibling Node): 같은 부모를 가진 노드들
- 리프 노드(Leaf Node): 자식이 없는 노드
- 내부 노드(Internal Node): 자식이 하나 이상 있는 노드
- 깊이(Depth): 루트에서 특정 노드까지의 간선 수
- 높이(Height): 특정 노드에서 리프 노드까지 가장 긴 경로의 간선 수
- 레벨(Level): 루트에서 특정 노드까지의 간선 수로, 깊이와 동일
- 차수(Degree): 노드가 가진 자식의 수
- 경로(Path): 한 노드에서 다른 노드로 가는 노드들의 순서
- 경로 길이(Path Length): 경로에 포함된 노드의 수
- 크기(Size): 특정 노드와 그 자손 노드들의 총 수
- 너비(Width): 특정 레벨에 있는 노드의 수
- 리프 수(Breadth): 트리 내 리프 노드의 총 수
- 거리(Distance): 두 노드 사이의 최단 경로에 있는 간선 수
- 차수(Order): 부모 노드가 가질 수 있는 최대 자식 수

---

## 트리의 특징

- 하나의 루트 노드와 0개 이상의 하위 트리로 구성
- 비선형 자료구조로, 순차적인 데이터 저장이 아님
- 재귀적인 구조를 가지며, 순환이 없는 무방향 그래프
- 모든 자식 노드는 하나의 부모 노드만을 가짐
- 노드가 n개인 트리는 항상 n-1개의 간선을 가짐

---

## 트리의 종류

- 편향 트리(Skew Tree): 모든 노드가 하나의 자식만을 가지는 트리
- 이진 트리(Binary Tree): 각 노드가 최대 두 개의 자식을 가지는 트리
- 이진 탐색 트리(BST): 왼쪽 자식은 부모보다 작고, 오른쪽 자식은 부모보다 큰 값을 가지는 이진 트리
- m원 탐색 트리(m-way Search Tree): 최대 m개의 자식을 가지는 탐색 트리
- 균형 트리(Balanced Tree, B-Tree): 높이 균형을 유지하는 m원 탐색 트리

---

## 트리의 사용 사례

- 계층적 데이터 저장: 파일 시스템 등에서 계층 구조를 표현할 때 사용
- 효율적인 검색: 삽입, 삭제, 검색 등의 연산을 효율적으로 수행할 수 있음
- 힙(Heap): 우선순위 큐를 구현할 때 사용되는 트리 구조
- 데이터베이스 인덱싱: B-Tree, B+Tree 등을 통해 빠른 데이터 검색을 지원
- 트라이(Trie): 문자열 검색을 위한 특수한 트리 구조
